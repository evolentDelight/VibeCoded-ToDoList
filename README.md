# VibeCoded-ToDoList
Vibe coded using the Cursor Agent on Auto

The purpose is to audit a basic app generated by the AI agent.

## Prompts
### Prompt #1:
```text
Build a complete to-do app using strictly JavaScript, React, Node.js, and Render for deployment. Also, implement a feature where a user can export and import to-do tasks in the form of CSV, JSON, or XML. 
```

---

A full-stack to-do app built with **JavaScript**, **React**, **Node.js**, and deployable on **Render**. Includes export/import of tasks in CSV, JSON, and XML formats.

## Features

- Add, complete, and delete tasks
- Export tasks to CSV, JSON, or XML
- Import tasks from CSV, JSON, or XML files
- PostgreSQL on Render (or file-based storage for local dev)
- Dark-themed UI

## Tech Stack

- **Frontend**: React (Vite)
- **Backend**: Node.js, Express
- **Database**: PostgreSQL (Render) or JSON file (local)
- **Deployment**: Render

## Local Development

### Prerequisites

- Node.js 18+
- (Optional) PostgreSQL for persistent storage

### Setup

1. Install dependencies:

```bash
npm run install:all
```

2. Start the backend (from project root):

```bash
npm run dev:backend
```

3. In another terminal, start the frontend:

```bash
npm run dev:frontend
```

4. Open [http://localhost:5173](http://localhost:5173)

The backend runs on port 3001. Without `DATABASE_URL`, it uses file-based storage in `backend/data/tasks.json`.

### With PostgreSQL

Create a PostgreSQL database (e.g. [Neon](https://neon.tech) or local) and set:

```bash
DATABASE_URL=postgresql://user:password@host:5432/dbname
```

## Deploy on Render

1. Push this repo to GitHub.
2. In [Render Dashboard](https://dashboard.render.com), click **New** → **Blueprint**.
3. Connect the repo. Render will detect `render.yaml`.
4. The blueprint creates:
   - A **Web Service** (Node.js app)
   - A **PostgreSQL** database (free tier)
5. Deploy. The app will be available at `https://<your-service>.onrender.com`.

### Manual Setup (without Blueprint)

1. Create a **PostgreSQL** database on Render.
2. Create a **Web Service** and connect your repo.
3. Set:
   - **Build Command**: `cd backend && npm install && cd ../frontend && npm install && npm run build`
   - **Start Command**: `cd backend && node server.js`
4. Add environment variable: `DATABASE_URL` = your PostgreSQL connection string (from Render).
5. Deploy.

## Export / Import

- **Export**: Use the CSV, JSON, or XML buttons to download all tasks.
- **Import**: Click "Import from file" and choose a `.csv`, `.json`, or `.xml` file. Imported tasks replace existing ones.

### File Formats

- **CSV**: `id,title,completed,created_at`
- **JSON**: Array of `{ id, title, completed, created_at }`
- **XML**: `<tasks><task><id>...</id><title>...</title><completed>...</completed></task></tasks>`
